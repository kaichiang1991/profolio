# i18n (Internationalization) Design

## Overview

Add bilingual support (Traditional Chinese / English) to the portfolio site, defaulting to Traditional Chinese.

## Approach

Manual lightweight i18n using React Context — no external i18n libraries.

## Architecture

- `LanguageContext` provides `locale` (`'zh'` | `'en'`), `setLocale`, and `t()` translation function
- Translation files: `src/i18n/zh.ts` and `src/i18n/en.ts` (typed nested objects)
- Default locale: `zh`
- Preference persisted in `localStorage` (key: `lang`)

## Translation File Structure

```ts
// src/i18n/zh.ts
export default {
  nav: { home: '首頁', projects: '作品集', experience: '經歷', contact: '聯繫' },
  hero: { greeting: '嗨，我是 Kai', subtitle: '...', viewProjects: '查看作品', skillTags: [...] },
  projects: { title: '作品集', subtitle: '...' },
  experience: { title: '經歷', subtitle: '...' },
  contact: { title: '聯繫', subtitle: '...' },
  footer: { rights: '版權所有' },
}
```

English file mirrors the same structure with English values.

## Language Toggle UI

- Navbar right side: `zh / EN` toggle button next to navigation links
- Simple text button, active language styled bold/dark

## Data Files

Project and experience data use per-locale fields:

```ts
interface Project {
  title: { zh: string; en: string }
  description: { zh: string; en: string }
  tech: string[]  // tech tags stay in English
  github?: string
  demo?: string
}
```

## Affected Components

- `src/i18n/` — new directory with context, types, translation files
- `src/components/Navbar.tsx` — nav link text + language toggle button
- `src/components/Footer.tsx` — copyright text
- `src/pages/Home.tsx` — all static text
- `src/pages/Projects.tsx` — all static text
- `src/pages/Experience.tsx` — all static text
- `src/pages/Contact.tsx` — all static text
- `src/data/projects.ts` — bilingual fields
- `src/data/experience.ts` — bilingual fields
- `index.html` — dynamic lang attribute (via context)
